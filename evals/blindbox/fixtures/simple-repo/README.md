# Simple Growth Workspace

This repository is intentionally sparse. The product direction is to improve the first-run experience for a tiny command-line task tracker.

Current facts:

- There is no implementation yet.
- The goal is to raise activated first-run users from 18% to 35%.
- The thing that must not be sacrificed is support quality: keep first-week support tickets at or below 12 per 100 activated users.
- The first delivery step should be small enough to verify locally.

Use the repo facts to create a Reverse Tornado loop before implementation.
