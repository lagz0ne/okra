# Learning Memory With No Eval Guard

Objective: `learning_memory_reuse_acceptance_rate >= 0.85`.

At run start the orchestrator automatically scans previous runs and loads prior learning as
candidate-only input. This is OKRA learning memory for the same project context key. It extracts
traps, avoidances, misconceptions, optimizations, and reusable candidate anti-goals with metric
thresholds and human ratification status.

Stress means time and DKR budget pressure. The orchestrator allocates DKR as the learning allocator
to decide whether to promote, fund, dry-run, veto, pause, re-aim, or raise cannot. If the objective
is unachievable under the current frame, cannot opens. Self-healing uses cannot, breaking,
pointless, and authority_drift flags for repair.

Evidence includes hashes, append-only ledger records, deterministic checks, and independent review.
`single_llm_truth_acceptance_count == 0`.

The artifact gives no baseline comparison gate for checker quality before accepting a memory rule.
